Detailed Description

template<int margin_, typename Value_, typename Index_, class IndexStorage_>
class tatami::DelayedSubsetSorted< margin_, Value_, Index_, IndexStorage_ >

Delayed subsetting of a matrix with sorted indices.

Implements delayed subsetting (i.e., slicing) on the rows or columns of a matrix, given a vector of sorted indices. This operation is "delayed" in that it is only evaluated on request, e.g., with row() or friends.

Template Parameters
margin_Dimension along which the subsetting is to occur. If 0, the subset is applied to the rows; if 1, the subset is applied to the columns.
Value_Type of matrix value.
Index_Type of index value.
IndexStorage_Vector containing the subset indices.

◆ dimension_preference()

template<int margin_, typename Value_ , typename Index_ , class IndexStorage_ >
std::pair< double, double > tatami::DelayedSubsetSorted< margin_, Value_, Index_, IndexStorage_ >::dimension_preference ( ) const
inlinevirtual
Returns
A pair containing the number of matrix elements that prefer row-level access (first) or column-level access (second).

This method is useful for determining the return value of prefer_rows() in combined matrices consisting of both row- and column-preferred submatrices. In such cases, the net preference can be determined based on the combined size of the submatrices for each preference.

For simpler matrices, the return value contains the total size of the matrix in one of the doubles and zero in the other.
