Detailed Description

template<typename Id_, typename Index_, class ChunkContents_>
class tatami::OracleChunkCache< Id_, Index_, ChunkContents_ >

Oracle-aware cache for chunks.

Template Parameters
Id_Type of chunk identifier, typically integer.
Index_Type of row/column index produced by the Oracle.
ChunkContents_Class that contains the contents of a single chunk.

Implement an Oracle-aware cache for chunks. This is typically used for Matrix representations where the data is costly to load (e.g., from file) and an Oracle is provided to predict future accesses. In such cases, the appropriate chunks of data can be loaded and cached once, such that all future requests will just fetch the cached data.

Fetch the next chunk according to the stream of predictions provided by the Oracle.

Template Parameters
Ifunction_Function to identify the chunk containing each predicted row/column.
Sfunction_Function to swap two chunks' contents.
Rfunction_Function to determine whether a chunk object is a non-mock instance.
Afunction_Function to allocate memory to a mock chunk for storage of chunk contents.
Pfunction_Function to populate zero, one or more chunks with their contents.
Parameters
identifyFunction that accepts an i, an Index_ containing the predicted row/column index. This should return a pair containing (1) the identifier of the chunk containing i, and (2) the index of row/column i inside that chunk.
swapFunction that accepts two ChunkContents_& and swaps their contents.
readyFunction that accepts a const ChunkContents_& and returns a boolean indicating whether it has already been allocated. This should return true for objects that have been used in allocate(), and false otherwise.
allocateFunction that accepts a single default-initialized ChunkContents_ object, and allocates sufficient memory to it in order to hold a chunk's contents when used in populate().
populateFunction that accepts two arguments, chunks_in_need and chunk_data. (1) chunks_in_need is a const std::vector<std::pair<Id_, Index_> >& specifying the chunks to be populated. The first Id_ element of each pair contains the chunk identifier, i.e., the first element returned by the identify function. The second Index_ element specifies the index of the ChunkContents_ in chunk_data in which to store the contents of each chunk. (2) chunk_data is a std::vector<ChunkContents_>& containing the cached chunk contents to be populated. This function should iterate over the chunks_in_need and populate the corresponding entries in chunk_data.
Returns
Pair containing (1) a pointer to a chunk's contents and (2) the index of the next predicted row/column inside the retrieved chunk.

Doing a linear scan across chunks to find the allocated but unused cache elements. This is the simplest and safest approach; trying to keep track of the unused caches would require a scan to fill a map/set anyway, and deleting an iterator from cache_exists only works if cache_exists actually contains all cache elements, which it might not be if we reached max_predictions without filling up the cache.

In any case, a linear scan should be pretty good for consecutive access; the first cache elements would be the oldest, so there wouldn't be any wasted iterations to find available cache elements.
